# Dashboard: Channels & Entities

Enhanced dashboard for duoduo that adds **Channels** and **Entities** tabs alongside the default Event Stream view. Retains the ATC dark theme.

## Features

### Event Stream (default tab)
- Real-time event log with auto-follow (LIVE/PAUSED toggle)
- Signal bar showing cortex sessions, job status, and subconscious partitions
- Tool calls, results, agent output with expandable markdown rendering
- Click agent output to expand full markdown; double-click any event for raw JSON
- Header shows aggregate cost, token count, and tool call stats

### Channels tab
- **Global Config** section: view/edit `config/feishu.md`, `memory/CLAUDE.md`, and `CLAUDE.md` (kernel)
- **Group Channels** section: lists all Feishu group channels with live status indicators (running/standby/offline)
- Each channel card shows the channel descriptor with rendered markdown, raw view toggle, and inline edit/save
- New channels auto-populate from `config/sample-descriptor.md` template
- Channel names resolved via `config/channel-id-map.json` (generated by `refresh-channel-map.py`)

### Entities tab
- Lists all entity files from `memory/index.md`
- Search/filter across entity names, display names, and descriptions
- Expandable cards with markdown rendering, raw view, and inline edit/save

## Installation

### 1. Copy the dashboard HTML

Replace the built-in dashboard with the enhanced version:

```bash
cp contrib/dashboard/dashboard.html bootstrap/dashboard.html
```

This is the only required step. The Channels and Entities tabs will work immediately using the duoduo RPC API (`channel.file.download`) for read access.

### 2. Generate channel name map (recommended)

The Channels tab shows Feishu channel IDs by default. To display human-readable group names, run the channel map generator:

```bash
python3 contrib/dashboard/scripts/refresh-channel-map.py
```

This scans `.aladuo/var/channels/` for `feishu-oc_*` directories, calls the Feishu API to resolve group names, and writes `aladuo/config/channel-id-map.json`.

**Requires**: `FEISHU_APP_ID` and `FEISHU_APP_SECRET` in `.env`. Uses `FEISHU_DOMAIN` to select between `open.feishu.cn` (feishu) and `open.larksuite.com` (lark).

### 3. Enable save-api (optional)

The Edit/Save buttons in the dashboard write files through a lightweight HTTP server running on the host. Without it, the dashboard is read-only.

```bash
python3 contrib/dashboard/scripts/save-api.py &
```

- Listens on `127.0.0.1:20234` (localhost only)
- Writes are restricted to an allowlist of path prefixes:
  - `aladuo/config/` — channel configs and descriptors
  - `aladuo/memory/entities/` — entity memory files
  - `.aladuo/var/channels/` — per-channel descriptors
- Path traversal is blocked via `os.path.realpath()` + prefix check
- CORS headers allow the dashboard (served from duoduo's built-in HTTP) to reach the API

### 4. Set up cron sync (optional)

To keep channel names and descriptors up to date automatically:

```bash
# Add to crontab:
*/5 * * * * cd /path/to/project && bash contrib/dashboard/scripts/sync-channels.sh >> /tmp/sync-channels.log 2>&1
```

This runs `refresh-channel-map.py` followed by `fill-empty-descriptors.py` every 5 minutes.

### 5. Fill empty descriptors (optional)

For channels that have a directory but only a stub descriptor (no `<channel-meta>` block), this script fills them from the `config/sample-descriptor.md` template:

```bash
python3 contrib/dashboard/scripts/fill-empty-descriptors.py
```

It reads `config/channel-id-map.json` for channel names and substitutes `{{CHANNEL_ID}}` and `{{CHANNEL_NAME}}` placeholders.

## Dependencies

| Dependency | Required | Purpose |
|---|---|---|
| Python 3 | Yes | Scripts (refresh-channel-map, save-api, fill-empty-descriptors) |
| Feishu credentials | For channel names | `FEISHU_APP_ID` + `FEISHU_APP_SECRET` in `.env` |
| inotifywait | No | Only needed by `image-resize-watcher.sh` (not included in this package) |

## How to revert

Replace the dashboard HTML with the original:

```bash
git checkout bootstrap/dashboard.html
```

Or simply remove the file and let duoduo regenerate its default dashboard on next start.

The scripts are standalone and do not modify duoduo internals. Removing them has no side effects.
